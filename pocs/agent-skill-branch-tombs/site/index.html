Branch Graveyard

A Claude Code skill that hunts down the branches your repo stopped touching, marks each with a headstone showing its last-touch date and author, and tells you which ones are safe to bury.

/branch-tombs

⬇ Install

Copies the skill into ~/.claude/skills/branch-tombs.

git clone <this repo> cd agent-skill-branch-tombs $ ./install.sh

☠ Uninstall

Removes the skill. Your reports are never touched.

$ ./uninstall.sh

What it digs up

🧊
Last touch & author
For every branch: the date of its last commit and who made it — straight from git refs, never guessed.
⏱
Age tiers
active, stale, abandoned, ancient — each headstone colored by how cold the branch has gone.
✓
Safe to bury
Stale branches already merged into the trunk, with the exact delete command and a copy-all button.
⚠
Unmerged graves
Old branches still holding commits not in the trunk — the ones to review before you delete.

Usage

/branch-tombsbury branches stale for more than 30 days (default)
/branch-tombs 60use a 60-day staleness threshold
/branch-tombs owner/repoclone a GitHub repo and build its graveyard

The skill writes a self-contained light-theme report to branch-graveyard-report/index.html in the current directory and opens it. It is read-only — it never deletes a branch; it only shows you the commands.
