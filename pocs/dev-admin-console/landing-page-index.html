What it does

Built for the moment you have seven servers and one question

Everything below is in the repository today.

Schema tree

Engine-shaped browsing

Foldable and lazy: schema→table→column, keyspace→table→column, key→field, prefix→key, topic→partition, index→field.

Editor

CodeMirror 6, live autocomplete

Highlighting and completions fed from the live schema of the connection you are on. ⌘↵ or Ctrl↵ runs the statement.

Paging

Server-side cursors

Cassandra paging state, Kafka offsets, Elasticsearch search_after, SQL offsets — each engine paged the way it actually pages.

Entity trace

One value, every connection

Search 42 and get the Postgres rows, the MySQL invoices, the Kafka messages, the Elasticsearch document and the Redis key on one timeline.

Federation

Cross-engine join

Up to five sources in one SELECT. A bounded hash join over native queries — not a query planner — so every guard still applies per side.

Discovery

Import running containers

Finds containers that look like a supported engine, detects the engine and credentials, and imports the ones you tick as a project.

Audit

Grouped by query

Paging through six pages reads as one entry. Denials are kept with their reason, so a blocked statement is evidence rather than a dead end.

Secrets

AES-256-GCM at rest

Connection passwords are encrypted, never returned by any API, never logged, and never present in an audit row or a screenshot.

AI

Your agent CLI writes the query

claude -p, codex exec or agy -p, chosen per user and stored in Postgres. Suggestions pass the same guard and never execute on their own.

Read-only

Three layers, no toggle to turn them off

A statement guard, a read-only connection inside a rolled-back transaction, and a SELECT-only database account. What each engine allows:

EngineAllowedRefused
CassandraSELECTall DML, DDL, TRUNCATE
Rediscommands the server itself flags readonlywrites, admin, EVAL — Lua can write
etcdget, get --prefix, rangeput, del, txn, compact
Kafkalist, describe, offsets, bounded consumeproduce, topic admin, offset resets
ElasticsearchGET/HEAD on _search, _count, _mapping, _catevery other verb, plus _bulk, _delete_by_query, _reindex even as GET

Kafka never joins a consumer group and never commits an offset. An integration test consumes 50 messages, then asserts a real consumer group's committed offsets are byte-identical before and after.
