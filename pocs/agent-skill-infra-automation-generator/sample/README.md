## About the Skill

All files in this directory were generated by the **infra-automation-generator** Claude Code skill. The skill scans a project, auto-detects the language, framework, ports, and dependencies, then generates production-ready infrastructure files based on user selections.

The skill works in 5 phases:

1. **Project Scanning** - Detects build files (`Cargo.toml`, `package.json`, `pom.xml`, `go.mod`, etc.), identifies the framework, finds the listening port, and discovers dependencies like databases, caches, and message brokers.
2. **User Choice Menu** - Presents options: Containerfile + podman-compose, Kubernetes manifests, Helm chart, OpenTofu modules (AWS), and Ansible playbooks.
3. **Follow-Up Questions** - Resolves conflicts (e.g., K8s vs Helm overlap) and gathers AWS pattern preferences for OpenTofu.
4. **Generation** - Produces all selected infrastructure files in dependency order with multi-stage container builds, health probes, resource limits, autoscaling, and proper networking.
5. **Summary** - Reports all generated files and provides next-step commands to deploy.

For this sample, the skill detected a Rust/Axum backend and a Node.js/Next.js frontend from the [agent-werewolf](https://github.com/diegopacheco/ai-playground/tree/main/pocs/agent-werewolf) source code and generated Containerfiles, podman-compose, Kubernetes manifests, a Helm chart, and OpenTofu AWS modules (VPC + ECS).
