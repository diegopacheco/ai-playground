Testing

One assert per test warning static

A test method has exactly one assertion.

One assert per test keeps failures specific and each test focused on a single behavior.

good

@Test
void registers() {
    assertNotNull(service.register("1", "Ann"));
}

bad

@Test
void registers() {
    User u = service.register("1", "Ann");
    assertNotNull(u);
    assertEquals("1", u.getId());
}

Design

A DAO is used by at most one service error semantic

No DAO is depended on by more than one service.

Sharing a DAO across services couples them and blurs data ownership.

good

class UserService {
    UserService(UserDao dao) { ... }
}
class AccountService {
    AccountService(AccountDao dao) { ... }
}

bad

class UserService {
    UserService(UserDao dao) { ... }
}
class AccountService {
    AccountService(UserDao dao) { ... }
}

Code

Never return null error static

Methods must not return null; return an empty object instead.

Null returns push NullPointerException risk onto every caller.

good

User findById(String id) {
    return store.getOrDefault(id, User.empty());
}

bad

User findById(String id) {
    if (store.containsKey(id)) {
        return store.get(id);
    }
    return null;
}
