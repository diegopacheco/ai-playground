# Custom Commands

Claude Code custom commands are a powerful way to extend the functionality of your Claude Code environment. They allow you to define your own commands that can be executed within the Claude Code interface, enabling you to automate tasks, integrate with external services, and enhance your workflow.

## Custom Commands Ideas

Code Quality & Analysis

1. /dead-code - Find and remove unused functions, variables, and imports
2. /complexity - Analyze code complexity and suggest simplifications
3. /dependencies - Audit and update project dependencies
4. /security-scan - Check for common security vulnerabilities
5. /type-coverage - Analyze TypeScript type coverage and add missing types

Documentation

6. /api-docs - Generate API documentation from code
7. /readme - Generate or update README.md based on codebase
8. /changelog-from-commits - Create changelog from recent git commits
9. /function-docs - Add JSDoc/docstrings to functions
10. /architecture-diagram - Generate ASCII architecture diagrams

Testing

11. /test-coverage - Run tests and analyze coverage gaps
12. /generate-tests - Create unit tests for selected files
13. /e2e-tests - Generate end-to-end test scenarios
14. /snapshot-tests - Create snapshot tests for components
15. /test-data - Generate realistic test fixtures

Refactoring

16. /extract-component - Extract reusable component from code
17. /modernize - Update code to latest language features
18. /split-file - Break large files into smaller modules
19. /rename-safe - Safely rename across entire codebase
20. /remove-duplication - Find and eliminate code duplication

Git & CI/CD

21. /pr-template - Create pull request with smart description
22. /commit-split - Split large changes into logical commits
23. /release-notes - Generate release notes from PR history
24. /ci-fix - Debug and fix CI/CD pipeline failures
25. /conflict-resolve - Help resolve merge conflicts

Performance

26. /performance-audit - Analyze bundle size and runtime performance
27. /optimize-images - Compress and optimize image assets
28. /lazy-load - Add lazy loading to components/routes
29. /cache-strategy - Implement caching strategies

Utilities

30. /boilerplate - Generate common patterns (API routes, components, etc.)
